Kennedy Cheroben — personal portfolio and design studio template showcasing frontend development, branding, and UI component work.

This repository is a Vite + React + TypeScript starter wired with Tailwind CSS and a shadcn-ui based component set. It contains example pages (portfolio, contact, boutique and electronics shop layouts) and a small UI library used across the site.

## Quick start (Windows PowerShell)

```powershell
# 1) Install dependencies
npm install

# 2) Start development server (hot reload)
npm run dev

# 3) Build for production
npm run build

# 4) Preview the production build locally
npm run preview
```

## Scripts

- `npm run dev` — start development server (Vite)
- `npm run build` — build production bundle
- `npm run preview` — locally preview production build
- `npm run lint` — run ESLint
- `npm test` — run tests (Vitest)

## Tech stack

- Vite
- React + TypeScript
- Tailwind CSS
- shadcn-ui (Radix primitives + Tailwind components)
- Framer Motion, react-router, sonner (toast), recharts, lucide icons

## Project structure (important folders)

- `src/` — application source
	- `assets/` — images and static assets
	- `components/` — shared components and UI primitives
		- `ui/` — shadcn-ui style primitives
	- `pages/` — page-level components (Portfolio, Services, Boutique, Electronics)
	- `lib/` — small utilities
	- `contexts/`, `hooks/` — app utilities
- `public/` — static files served as-is
- `index.html` — app entry page

## Development notes and assumptions

- The project expects Node.js and npm installed. Tested with recent Node LTS.
- `package-lock.json` can be (re)generated by running `npm install` if you need an exact lockfile for CI.
- Some scaffold/demo references were removed from the README to keep the repository neutral and portfolio-focused.

## Deployment

Deploy to any static hosting that supports Vite builds (Vercel, Netlify, Cloudflare Pages). Typical flow:

1. Push to GitHub.
2. Configure the host to run `npm install` and `npm run build`.
3. Serve the `dist/` output.

## Linting and testing

- Lint: `npm run lint` (ESLint configured in repo)
- Tests: `npm test` (Vitest)
